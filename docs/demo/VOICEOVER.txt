A green build is not a shippable game. PlayReceipt is an evidence gate for AI-built games. It turns reliability, balance, accessibility, and human-play evidence into a stable release receipt. The key is honesty: the tool returns Pass, Repair, Human Review, or Unverified. It never hides missing evidence behind one synthetic score.

The default case is a real, read-only project snapshot. Two thousand seeded simulations completed with zero crashes and zero stalls. Those are real passes. But player strategy changes the win rate by only eight point two percentage points, the best intentional policy performs slightly worse than random, reduced motion is missing, and no human has calibrated fun. So the overall verdict is Repair. Every row shows the observed evidence, the threshold, its provenance, and the smallest credible repair.

Now the judge can audit new evidence, not just watch a showcase. Paste a JSON bundle with no account and issue a receipt. The supplied sample passes seven machine gates. The receipt is exact, copyable, and downloadable. A four-stage trail shows what was ingested, how the rules judged it, whether a repair plan exists, and where automation must stop. This one stops at Human Review. A simulation can verify reliability and decision leverage. It cannot certify fun.

Under the hood, the browser, API, command line, and GitHub Action call the same audit engine. Identical evidence produces an identical SHA-two-fifty-six receipt ID. Repair and Unverified fail continuous integration; Human Review stays successful with a warning. Eleven regression tests cover the adapters and abuse boundaries. A repository-local Codex skill can preserve the source, execute the repair plan, and rerun the same seeds.

PlayReceipt is new OpenAI Build Week work in the Developer Tools track. Codex and GPT-five-point-six shaped the product contract, implementation, adversarial tests, and browser verification. Existing game evidence is isolated and hashed; the product, simulator, interface, action, tests, and receipts are new.
