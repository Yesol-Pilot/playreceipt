A green build is not a shippable game. PlayReceipt is an evidence gate for AI-built games. It turns reliability, balance, accessibility, and human play evidence into a stable release receipt. The key is honesty: the tool returns Pass, Repair, Human Review, or Unverified. It never hides missing evidence behind one synthetic score.

The default case is a real, read-only project snapshot. Two thousand seeded simulations completed with zero crashes and zero stalls. Those are real passes. But player strategy changes the win rate by only eight point two percentage points, the best intentional policy performs slightly worse than random, reduced motion is missing, and no human has calibrated fun. So the overall verdict is Repair. Every row shows the observed evidence, the threshold, its provenance, and the smallest credible repair.

Now we switch to the deterministic repaired sandbox. The same rules and the same seeded workflow produce a twenty point two percentage point strategy signal. Reduced motion is present. Seven machine-verifiable gates pass, with zero repairs and zero missing inputs. But PlayReceipt still stops at Human Review. A simulation can verify reliability and decision leverage. It cannot certify fun.

Under the hood, identical evidence produces an identical SHA-two-fifty-six receipt ID. The command line uses meaningful exit codes for agents and continuous integration. Seven regression tests cover missing evidence, stable receipts, the documented command forms, and the live HTTP dashboard. A repository-local Codex skill can normalize evidence, run the audit, preserve the source project, and rerun the same seeds after a repair.

PlayReceipt is new OpenAI Build Week work in the Developer Tools track. Existing game evidence is isolated and hashed; the product, simulator, interface, tests, and receipts are new. Independent review found defects, the defects became regression tests, and final review passed at ninety out of one hundred.
